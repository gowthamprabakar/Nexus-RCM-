# NEXUS RCM ‚Äî Design Context

## Target Audience
- **Primary:** Revenue Cycle Management (RCM) specialists at mid-market US healthcare billing teams (20-200 FTEs). Specialized roles: denial analysts, appeal writers, patient access coordinators, collections agents, coding auditors, CFOs.
- **Context:** Working 6-8+ hour dashboard shifts. Heavy data density. Claims volume: 500K+ historical, thousands of daily decisions.
- **Pain points:** Previous tools feel like Excel-on-the-web ‚Äî slow, dated, sterile. Users are drowning in noise, want AI-prioritized worklists.

## Use Cases (jobs-to-be-done)
1. **Detect** denials fast (Denial Command Center) ‚Äî find the $2M at-risk this week
2. **Prevent** denials before submission (Claim Readiness, Prevention Engine, Patient Access, Coding Optimizer, Automation Rules) ‚Äî catch missing auth, bad codes, eligibility gaps
3. **Recover** lost revenue (Collections, AR Aging, Cash Flow, ERA Processing) ‚Äî dunning sequences, settlement offers, auto-dialer
4. **Predict** future revenue (Forecast, Simulation, Graph Explorer, LIDA AI) ‚Äî 90-day cash projections, payer behavior twins, NL queries
5. **Optimize** operations (Automation Rules, Coding Optimizer)

## Brand Personality / Tone
- **Primary vibe:** "Mission control for RCM" ‚Äî Bloomberg-terminal-for-healthcare confidence. Dense, fast, decisive. Dark-theme-first. Surgical not playful.
- **Industrial / utilitarian** with **editorial** hierarchy ‚Äî clean typography carries gravitas, data-density earns trust.
- **Feels like:** Linear + Stripe + Bloomberg Terminal + modern SaaS RCM = serious, trustworthy, premium.
- **Not:** Cute. Not gamified. Not "friendly chatbot" feel. Not childish emoji-heavy.

## Technical Constraints
- **Stack:** React 18 + Vite, FastAPI, PostgreSQL, Tailwind CSS with `th-*` design tokens (CSS variables in `frontend/src/index.css`), Recharts for charts
- **Theme system:** `th-*` classes resolve to CSS variables ‚Äî supports light/dark dual-theme. Existing tokens: `th-heading`, `th-secondary`, `th-muted`, `th-surface`, `th-surface-raised`, `th-surface-overlay`, `th-border`, plus semantic `primary`, `emerald`, `rose`, `amber`, `violet`, `cyan` accents.
- **Accessibility:** AA minimum (4.5:1 contrast). Healthcare = regulated, must be keyboard-navigable, screen-reader friendly.
- **Performance:** Pages must render first paint < 1s even with 2.5M rows in DB. API calls are aggregation endpoints.

## Differentiation ‚Äî What should be UNFORGETTABLE
1. **Density without noise** ‚Äî show 10x more data than competitors but users can still scan in 5 seconds. Achieved via typographic hierarchy, spacing discipline, and hairline dividers instead of boxes-within-boxes.
2. **AI-first** ‚Äî every page has an "AI Intelligence" narrative band at the top written by Ollama/Qwen3. Feels like having a CFO-chief-of-staff summarizing the situation.
3. **Scan-first data viz** ‚Äî number comes first at 32-48px, label comes second at 10-12px uppercase. Trend badge (+2.3%, -0.5%) is tiny but always present for every metric.
4. **Motion is restrained** ‚Äî subtle `hover:-translate-y-0.5` on cards, no spring bounces, no 1.5s page transitions. ADTP: animations ‚â§ 200ms, mostly ease-out.

## Anti-goals (DO NOT)
- Do NOT add gradients-on-gradients or color-on-color "glow" effects
- Do NOT use bounce or elastic easing
- Do NOT use clip-art medical icons (stethoscope, cross-plus, nurse cap) ‚Äî this is finance-for-healthcare, lean on material-symbols-outlined
- Do NOT use serif display fonts ‚Äî we want modern sans (Inter / system-ui) with strong weight contrast
- Do NOT use "playful" rounded chunky shapes ‚Äî corners are `rounded-lg` max, sharp is OK
- Do NOT hide critical numbers behind hover states ‚Äî density means visible by default
- Do NOT add emojis as decoration (they're OK as sidebar section markers like "üõ° Prevent" but nowhere inside content)
